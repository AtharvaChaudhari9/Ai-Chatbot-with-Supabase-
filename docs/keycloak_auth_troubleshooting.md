## 2. The Current Issue: Redirect Loop on Login
*   **Symptom:** After logging in, the browser redirects back to the landing page `/` (Get Started) in a continuous loop.
*   **Root Cause:** To handle the unstable Keycloak UUIDs generated by the pairwise subject configuration, we implemented a deterministic UUID generator and updated `token.sub` inside NextAuth's `jwt` callback:
    ```typescript
    // In auth.ts:
    token.sub = getDeterministicUuid(stableKey);
    ```
    NextAuth v5 uses the decrypted JWT's `token.sub` internally to validate OIDC session signatures and manage middleware access. Overwriting/mutating `token.sub` directly inside the `jwt` callback corrupts NextAuth's internal handshake validation. As a result, NextAuth thinks the session is invalid, rejects it, and triggers the middleware to redirect the browser back to `/` in a loop.
